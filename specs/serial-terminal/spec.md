# Spec: 多人共享串口终端

## 摘要

HubTerm 用户可以从节点详情页直接连接 Agent 上报的串口，并在浏览器中进行实时交互。一个串口在同一时间只建立一个物理连接，但允许多人进入同一个共享会话。会话包含一个可输入的主控用户和多个只读观察者，管理员可以转移主控权或移除参与者。

## 用户故事

- 作为管理员或操作员，我希望从串口列表直接连接设备，以便无需登录节点主机即可操作串口设备。
- 作为协同人员，我希望加入已有的串口会话并实时查看输出，以便共同排障。
- 作为管理员，我希望编辑并保存每个串口的通信参数，以便后续连接使用正确配置。

## 验收标准

- [ ] 每个串口行显示“连接”“新标签页连接”“编辑”操作，布局参考现有 SSH 配置操作列。
- [ ] 离线节点或离线串口不能发起连接，并明确说明原因。
- [ ] 编辑操作可以保存波特率、数据位、校验位、停止位和流控设置。
- [ ] 编辑操作可以设置可选的串口别名；列表和共享终端优先显示别名，同时保留真实设备路径。
- [ ] 首次使用默认参数为 `115200 / 8 数据位 / 无校验 / 1 停止位 / 无流控`。
- [ ] 参数按“节点 + 串口”保存，并对所有有权限的用户共享。
- [ ] 连接成功后进入共享终端；发起者成为主控，可以输入和调整终端大小。
- [ ] 其他用户进入同一串口时加入已有会话，默认只读，不会重复打开物理串口。
- [ ] 管理员可以转移主控权或移除参与者，只有当前主控可以向串口写入数据。
- [ ] 最后一名参与者离开后关闭物理串口并释放占用状态。
- [ ] 串口被拔出、Agent 离线、参数无效或打开失败时，用户看到可操作的错误信息，并可以返回或重试。
- [ ] 重复点击连接不会创建重复会话。
- [ ] 串口输出和输入保持原始字节内容，不进行文本编码转换。

## 用户体验

### 正常流程

1. 用户打开在线节点的详情页，在串口列表中找到目标端口。
2. 用户可以点击“编辑”检查或修改串口参数。
3. 用户点击“连接”在当前页面进入终端，或点击“新标签页连接”在新页面进入终端。
4. 系统立即显示连接中状态；成功后显示主控或观察者身份。
5. 主控输入会发送到串口，所有参与者实时看到串口输出。
6. 已有会话存在时，新用户直接加入该会话，不再次打开串口。

### 状态

- **加载中：** 串口列表加载时显示明确的加载状态，操作按钮不可用。
- **空状态：** 节点没有串口时显示“未发现串口”，并提示检查 Agent 和设备连接。
- **错误：** 加载或连接失败时显示具体原因，并提供重试或返回操作。
- **成功：** 连接成功后显示端口名、串口参数、会话状态和当前角色。
- **部分可用：** 某些端口离线时仍展示其他在线端口；离线端口保留信息但禁用连接。
- **离线：** Agent 离线或连接中断时终端停止输入，显示断线原因，不自动创建新会话。

### 边界情况

- 两名用户同时连接同一空闲串口时，只允许创建一个物理连接，另一名加入同一会话。
- 两名管理员同时修改参数时，以最后一次成功保存为准，界面显示最新值。
- 串口被其他系统进程占用时，不覆盖占用，显示端口忙及处理建议。
- 用户快速重复点击或网络缓慢时，连接按钮保持忙碌状态，避免重复提交。
- 串口名称较长时完整值可查看，操作按钮仍保持可用。
- 别名为空时回退显示真实设备路径；别名最多 128 个字符，保存时移除首尾空格。
- 无主控用户时，由管理员指定新主控；观察者不会自动获得写权限。

## 不在本次范围

- 自动重连或 Agent 重启后恢复旧会话。
- 串口数据录制、回放和下载。
- RS-485 专用方向控制等设备厂商扩展参数。
- 自动识别设备所需波特率或协议。
- 同一个物理串口建立多个独立会话。

## 非功能要求

- **权限：** 管理员和操作员可以创建串口会话；只有当前主控可以写入；普通观察者只能读取。
- **响应：** 点击操作后 100 毫秒内显示进行中反馈，串口数据以低延迟持续显示。
- **可靠性：** 浏览器异常关闭、Agent 断线或串口读取失败时必须释放会话和端口状态。
- **安全：** 服务端必须校验节点、串口、会话和用户角色，不能仅依赖按钮禁用。
- **可访问性：** 操作可通过键盘完成，按钮有清晰标签和焦点状态，错误信息不只依赖颜色。
- **可观测性：** 记录串口打开、关闭、失败、主控转移和异常断线事件，但不记录串口数据内容。
- **界面语言：** 沿用当前中文界面和现有组件风格。

## 待确认假设与风险

- 串口参数作为节点级配置共享，而不是每个用户单独保存。
- 第一版支持 Agent 能运行的桌面系统；不同系统的串口能力差异需要在实现验证中确认。
- 多人共享复用现有“主控/观察者”交互，串口会话不会另建一套权限模型。
